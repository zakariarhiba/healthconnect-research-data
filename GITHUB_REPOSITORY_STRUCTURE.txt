DIRECTORY STRUCTURE:

healthconnect-research-data/
│
├── README.md                              ✅ COMPREHENSIVE GUIDE

│   ├─ Data files overview
│   ├─ How to use the data
│   ├─ Citation information
│   └─ Email contact for implementation code
│
├── SIMULATION_RESULTS.csv                 ✅ MAIN DATA FILE (43.2M events)
│   └─ Columns: timestamp, patient_id, HR, SpO2, temp, latency_ms, alert_type
│
├── ARCHITECTURE_COMPARISON.xlsx           ✅ EXCEL WORKBOOK
│   ├─ Sheet 1: Performance Metrics
│   ├─ Sheet 2: Scalability Analysis
│   ├─ Sheet 3: Requirements Satisfaction (18/18 matrix)
│   ├─ Sheet 4: Cost Analysis
│   └─ Sheet 5: Clinical Impact Analysis
│
├── TEST_CONFIGURATION.md                  ✅ TEST SETUP DOCUMENTATION
│   └─ Hardware specs, network, test parameters
│
├── RESULTS/                               ✅ ANALYSIS CSV FILES
│   ├── latency_distribution.csv           (p50, p95, p99 for each architecture)
│   ├── performance_metrics.csv            (CPU, memory, throughput, uptime)
│   ├── architecture_comparison.csv        (18 requirements vs 4 architectures)
│   └── vital_signs_sample.csv             (First 1000 events as sample)
│
├── CONFIGURATIONS/                        ✅ TEST ENVIRONMENT SPECS
│   ├── test_environment.txt               (Hardware, network, test duration)
│   ├── kafka_config.txt                   (Broker configuration parameters)
│   ├── spark_config.txt                   (Streaming parameters)
│   └── test_parameters.txt                (Vital signs distribution, anomalies)
│
├── ANALYSIS/                              ✅ STATISTICAL ANALYSIS
│   ├── statistical_analysis.txt           (ANOVA, Tukey HSD, CI)
│   ├── requirements_matrix.txt            (18 requirements satisfaction)
│   └── clinical_impact_analysis.txt       (Latency vs mortality implications)
│
├── LICENSE                                ✅ RESEARCH DATA LICENSE
│   └─ Academic use only, no commercial deployment
│
└── .gitignore                             (No sensitive data)
